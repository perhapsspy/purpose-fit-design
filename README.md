# Purpose-Fit Design

[한국어](README.md) | [English](README.en.md)

## 요약

새 기능을 설계하거나 기존 컴포넌트의 재사용 여부와 임시 구현 방향을 정할 때, 사용자와 도메인의 목적, 제약, 현재까지 확인된 근거, 검증 가능한 성공 조건을 먼저 봅니다. 필요한 만큼만 대안을 비교하고 잠정 방향, 작은 확인 방법, 또는 결정을 막는 질문 하나를 남깁니다.

## 빠른 시작

**설치**

```bash
npx skills add perhapsspy/purpose-fit-design
```

혹은 `skills/purpose-fit-design` 폴더를 에이전트 스킬 디렉터리에 직접 복사합니다.

**바로 사용**

```text
$purpose-fit-design 이 새 검색 기능의 방향을 사용자 목적과 성공 조건에 맞춰 정해줘.

$purpose-fit-design 이 기능을 빠르게 임시 구현할 때 지켜야 할 제약과 검증 방법을 정해줘.

$purpose-fit-design 새 컴포넌트가 필요한지 기존 것을 그대로 써도 되는지 판단해줘.
```

## 이런 때 사용

- 새 기능의 설계나 구현 방향을 코딩 전에 정할 때
- 기존 구현을 재사용할지 새로 만들지 판단할 때
- 빠른 임시 구현의 범위, 검증과 되돌림 방법을 정할 때
- 기존 기본값이나 명시적으로 제외한 개념이 목적과 제약에 영향을 줄 때
