# 스킬 방향

## 목적

- 초기 설계나 구현 계획의 방향을 사용자와 도메인의 목적, 명시된 제약, 현재까지 확인된 근거, 검증 가능한 성공 조건에 맞춰 정한다.

## 유지할 것

- 목적, 제약, 근거와 관찰 가능한 성공을 판단의 중심에 둔다.
- 기존 구현과 선례는 자동 기준이 아니라 근거로 다룬다.
- 중요한 선택에서만 대안을 비교하거나 작은 확인 방법을 찾는다.
- 사용자 정정과 제외한 개념을 지키고, 탐색만으로 구현 범위를 넓히지 않는다.
- 작은 작업은 한 문단으로 끝낼 수 있게 유지한다.

## 피할 것

- 모든 재사용을 의심하거나 기존 구현을 거부하는 방향.
- 기준 소유자 확인, 의미 경계 설계와 코드 구조화를 이 스킬 안에서 수행하는 방향.
- 특수 사례와 실험 기록을 `SKILL.md`에 계속 추가하는 방향.

## 수정 기준

- 실제 작업에서 누락, 과잉 호출, 늦은 이관이 반복될 때만 배포된 스킬을 수정한다.
- 검증되지 않은 안정성 주장이나 운영 환경에서 검증됐다는 표현은 넣지 않는다.
- 한국어 문서는 영문 기본 스킬의 의미를 보존하되 자연스럽고 직접적으로 쓴다.
- 저장소 운영 문서는 스킬 사용 계약을 대신하지 않는다.
